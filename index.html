Job Code Check — How To Use

    
This tool checks whether a week's HotSchedules schedule covers all the placement slots — the ideal staffing template for each day type. It tells you what's fully covered, what's partially covered, and what's missing entirely.


    

      
1

      

        
Open the Daily Roster Report in HotSchedules

        
In HotSchedules, navigate to the Daily Roster Report — the web-based version, not the PDF. Set the date picker to the first day of the week you want to check. Make sure the Schedules filter at the top shows all schedules selected.

      

    


    

      
2

      

        
Open the browser console

        
Press F12 on your keyboard (or right-click anywhere on the page and choose Inspect, then click the Console tab at the top of the panel that opens).

      

    


    

      
3

      

        
Run the Full Week script

        
Go to the Console Scripts tab in this tool and click Copy next to the Full Week script. Switch back to HotSchedules, click inside the console, paste, and press Enter.

        
The script will automatically step through Monday to Saturday, scrape each day, and copy the full week to your clipboard when done. Watch the console for progress — it takes about 15–20 seconds. You'll see a green "Done!" message when it's finished.

        
Keep the HotSchedules tab active while the script runs — switching away may slow it down.

      

    


    

      
4

      

        
Paste into the Run Check tab

        
Switch to the Run Check tab in this tool. Click into the large text box on the left and paste (Cmd+V or Ctrl+V). Then click Run Check.

        
If the clipboard copy was blocked, the data will be printed in the console — select it all, copy, and paste manually.

      

    


    

      
6

      

        
Read the results

        
Results are organized by day, then by section (FOH, FOH Leadership, BOH, BOH Leadership). Each placement slot shows one of three statuses:

        

          
✓ OKThe slot is fully covered for the day.

          
~ PartialSome coverage exists but there's a gap — the row will show exactly which time window is uncovered.

          
✗ MissingNo scheduled shifts cover this slot at all.

        

        
At the bottom of each day you'll also see Extra Coverage (shifts scheduled beyond what any slot requires — potential hours to trim) and Unmatched Shifts (shifts that don't overlap any placement slot — usually Other Store shifts, which are expected and can be ignored).

      

    


    

      
7

      

        
Updating the placement (a few times a year)

        
When the placement changes, open the Placement tab in Google Sheets, go to File → Download → Comma Separated Values (.csv), then come back here and click the relevant day-type button under Import CSV quick-load on the Run Check tab (Mon–Wed, Thu–Fri, or Sat). The new placement will save automatically and be used for all future checks.

        
Your placement is saved in the browser on this computer. If you switch computers or clear your browser data, you'll need to re-import the CSVs.

      

    


    

      
Quick Reference — Gap Tolerance

      
The Gap Tolerance setting (top right of Run Check) controls how many minutes of uncovered time is still considered "OK." The default is 15 minutes — so if one person's shift ends at 2:00p and the next starts at 2:10p, that gap won't trigger a warning. Set it to 0 for strict back-to-back checking.
